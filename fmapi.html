FileMaker Data API에 관하여

개요

FileMaker ® Data API는 웹 서비스가 호스트 된 데이터베이스의 데이터에 접근 할 수 있도록하는 API (Application Programming Interface)입니다. 이 API는 REST (Representational State Transfer) 아키텍처를 준수하므로 FileMaker Data API는 REST API입니다.

웹 서비스 또는 응용 프로그램은 FileMaker Data API를 호출하여 호스팅 된 데이터베이스에 액세스하기위한 인증 토큰을 얻은 다음,이 호출을 사용하여 레코드를 생성하고 레코드를 업데이트하며 레코드를 삭제하고 찾기 요청을 수행합니다.

FileMaker Data API는 특정 프로그래밍 언어 형식과 독립적이므로 REST API와 함께 일반적으로 사용되는 텍스트 형식 인 JSON (JavaScript Object Notation)으로 데이터를 반환합니다.

이 안내서는 다음과 같은 경험이 있다고 가정합니다.

  • FileMaker Pro Advanced를 사용하여 데이터베이스 생성하기. FileMaker Pro Advanced 데이터베이스 디자인의 기본 사항 및 필드, 관계, 레이아웃, 포털 및 컨테이너의 개념을 이해해야합니다. FileMaker Pro 고급 도움말을 참조하십시오 .
  • FileMaker Server를 사용하여 데이터베이스 호스팅. FileMaker Server를 배포하고 호스트 된 데이터베이스에 접근 할 수있게하고 FileMaker Server 관리 콘솔을 사용하여 호스트 된 데이터베이스를 모니터링하는 방법을 이해해야합니다. FileMaker Server 도움말을 참조하십시오 .
  • JSON 형식의 데이터로 POST, GET, PATCH 및 DELETE 메소드를 호출하는 웹 서비스 또는 서버 측 애플리케이션에서 REST API 사용 선택한 프로그래밍 언어 또는 도구를 사용할 수 있습니다.

FileMaker Data API를 사용하려면 다음 단계를 따르십시오.

  1. FileMaker Pro Advanced를 사용하여 FileMaker Data API에 액세스 할 수 있도록 데이터베이스를 준비하십시오. 데이터베이스를 작성하거나 기존 데이터베이스를 준비 할 수 있습니다. FileMaker Data API 접근을위한 데이터베이스 준비를 참조하십시오 .
  2. 호스팅 된 데이터베이스에서 레코드를 찾고 생성, 편집 및 삭제하기 위해 FileMaker Data API 메소드를 호출하는 코드를 작성하십시오. FileMaker Data API 호출 쓰기를 참조하십시오 .
  3. FileMaker Data API 액세스가 활성화 된 FileMaker Server를 사용하여 솔루션을 호스팅하십시오. FileMaker Data API 솔루션 호스팅을 참조하십시오 .
  4. FileMaker Data API 액세스가 올바르게 작동하는지 테스트하십시오. FileMaker Data API 솔루션 테스트를 참조하십시오 .
  5. 관리 콘솔을 사용하여 호스팅 된 솔루션을 모니터링하십시오. FileMaker Data API 솔루션 모니터링을 참조하십시오 .
